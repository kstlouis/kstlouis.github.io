Sidebar: code segments on web pages

Problem:

Sometimes you want to actually type out some code and have it displayed on your page, say for explanatory or demo purposes, but make it look different, so the reader knows explicitly that this is a code segment.

In such a case, you likely will want to type out specific characters that HTML relies on to make code work. You can't use it without breaking anything.

Solution:

The <code> tag and HTML Entities

HTML Entities are specific characters that HTML relies on, like the "less than" (<) and "greater than" (>) symbols. The ampersand (&) is another. These are called reserved characters. A list of them, and how to type them out properly, can be found here.

The <code> tag, meanwhile, is a phrase tag that defines a piece of computer code, and will format it accordingly. You'll often use it in conjunction with reserved characters.

You can and often will it in conjuction with hyperlinks as well. For instance, if you wanted to learn more about specific things that we've previously covered, you can check out the MDN pages for <header>, <p>, <img>, and <table> by clicking on their respective (and now slightly nicer-looking) links. They still don't look that great, but CSS will likely help with that later down the road.
